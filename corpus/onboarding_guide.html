Getting started

This guide walks through the steps to start storing objects with NimbusStore.

Step 1: create an account

Sign up with your email address and verify it via the confirmation link NimbusStore sends you. Verification is required before you can create a bucket.

Step 2: create your first bucket

From the console, choose "Create bucket" and pick one of the three available regions: us-east-nimbus-1, us-west-nimbus-2, or eu-central-nimbus-1. The region cannot be changed after the bucket is created.

Step 3: generate an API key

Go to Settings, then API Keys, and generate a new key. Remember that keys expire 12 hours after issuance, so long-running scripts should handle re-authentication.

Step 4: upload your first object

Using the CLI: nimbus cp file.txt nimbus://mybucket/file.txt. Objects over 100 MB are uploaded automatically via multipart upload by the CLI; no extra flags are needed.

Step 5: optional - set a lifecycle rule

If you want objects to automatically move to the cheaper cold storage tier after a period of inactivity, configure a lifecycle rule on the bucket. By default, NimbusStore already auto-archives objects to cold storage after 90 days of inactivity even without a custom rule.
